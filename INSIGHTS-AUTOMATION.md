# Insights Automation — bi-weekly auto-researched articles

The Insights section is sourced from **Sanity CMS**. A Vercel Cron job runs every 2 weeks,
researches current AI/automation topics for SA SMEs with Claude + web search, and publishes
1–2 original, cited articles — with guardrails so nothing low-quality goes live unreviewed.

## How it works
```
Vercel Cron (09:00 on the 1st & 15th)
  → GET /api/cron/insights        (Authorization: Bearer CRON_SECRET — Vercel adds this automatically)
     → lib/insights/generate.ts   Claude (claude-opus-4-8) + web_search → 1 article, zod-validated
     → quality gate               ≥3 real source URLs + confidence ≥ 0.7 + not a duplicate
          pass → publish to Sanity   |   fail → save as Sanity DRAFT (never shown on the site)
     → email INSIGHTS_NOTIFY_EMAIL with a View + one-click Unpublish link
Insights pages read PUBLISHED posts from Sanity (lib/insights/getInsights.ts).
Cover image is chosen by category (lib/insights/categoryImages.ts).
```

## Key files
| File | Role |
|---|---|
| `sanity/schemaTypes/post.ts` | `post` schema + new fields: category, excerpt, readTime, sources, autoGenerated |
| `sanity/lib/writeClient.ts` | server-only Sanity client (needs `SANITY_API_WRITE_TOKEN`) |
| `lib/insights/getInsights.ts` | GROQ reads for the list + article pages |
| `lib/insights/generate.ts` | Claude + web_search generation, zod-validated output |
| `lib/insights/portableText.ts` | Section[] → Portable Text (shared by bot + seed) |
| `lib/insights/categoryImages.ts` | category → curated cover image |
| `lib/insights/notify.ts` | publish/draft email (nodemailer / SMTP) |
| `app/api/cron/insights/route.ts` | the bi-weekly orchestrator + quality gate |
| `app/api/insights/unpublish/route.ts` | one-click take-down (HMAC-signed link) |
| `scripts/seed-insights.ts` | one-time import of the 8 legacy articles into Sanity |

## One-time setup
1. Create `SANITY_API_WRITE_TOKEN` (Editor) in Sanity manage; set the env vars above (locally in
   `.env.local`, and in Vercel project settings).
2. Seed the existing articles into Sanity:
   ```
   npx tsx scripts/seed-insights.ts
   ```
3. Deploy. Vercel registers the cron from `vercel.json` (`0 9 1,15 * *`). Confirm the plan includes
   Cron Jobs.

## Testing the bot manually (before the first scheduled run)
```
curl -H "Authorization: Bearer $CRON_SECRET" https://<your-deployment>/api/cron/insights
# or locally:  curl -H "Authorization: Bearer $CRON_SECRET" http://localhost:3000/api/cron/insights
```
Then check `/studio` for the new post (or draft), the notification email, and `/insights`.

## Guardrails (auto-publish safety)
- Mandatory **real, cited sources** (≥3 working URLs) — invented stats/sources are rejected by the prompt + zod.
- **Original synthesis** only (no copied text), in Maru's evidence-led, no-hype voice.
- Below the quality bar (confidence < 0.7 or thin sources) ⇒ saved as a **draft**, never published.
- **Email on every run** with a one-click **unpublish** link; drafts link to the Studio for review.
- Dedupe against existing posts by slug.

To tune: `CONFIDENCE_THRESHOLD` / `MIN_SOURCES` in `app/api/cron/insights/route.ts`; voice and
output contract in `lib/insights/generate.ts`; swap cover images in
`public/images/insights/categories/`.
